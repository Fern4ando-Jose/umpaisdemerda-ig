0 · Insumo — curadoria de pauta (manual, à parte)

Ler o que o país aprontou na semana

Fora do fluxo automático: um script puxa as manchetes da semana em 4 frentes (Executivo, Legislativo, Judiciário, Crime/narco) só como insumo humano. Não publica nada; ajuda a calibrar os temas. A régua apartidária vale: nada de nomes/partidos.

Código
scripts/coletor-pauta.mjs
Conta/serviço
Google News RSS (público, sem chave)
trafega: ideias de tema (curadoria) → alimenta a régua de THEMES
1 · Gatilho — o relógio dispara sozinho

O cron acorda o pipeline no horário

Nos picos do Instagram (BRT), o agendador dispara um post. Plano previsto: ~2 carrosséis/dia + 3 Reels de vídeo/dia + 1 Reel clássico. O run (0..5) vem da expressão cron, e define o tom (manhã/tarde/noite) e a vaga do dia.

Infra desta peça grátis
Host
GitHub Actions (cron)
Rota/comando
.github/workflows/instagram-posts.yml · instagram-reels.yml · instagram-reels-classic.yml
Código
workflows YAML → chamam a API do site via secrets.SITE_URL (domínio agnóstico; sem DR) ⚠ Fase 2: repo/secrets UPM a criar
Conta/serviço
GitHub (repo desta automação) ⚠ Fase 2: repo/secrets UPM a criar
trafega: run + horário → GET /api/publish
2 · A pauta do dia — escolha do tema

Rotação dos 29 temas + trava anti-repetição

O sistema escolhe 1 dos 29 temas (4 pilares: servidão voluntária · a casta · o Estado que rouba · liberdade), rodando sem repetir e pulando o que já saiu nos últimos 7 dias em qualquer formato (Reel ou carrossel).

Infra desta peça grátis
Host
App Next.js (server, edge/serverless)
Rota/comando
GET /api/publish (?run= & ?preview=)
Código
src/app/api/publish/route.ts (THEMES) + src/lib/rotation.ts + run-ledger.ts
Conta/serviço
Neon Postgres (livro-razão anti-dup) ⚠ Fase 2: banco UPM a provisionar
trafega: tema + slot (tom)
3 · Pesquisa de apoio (grátis)

Puxa contexto factual sem pagar

Antes de escrever, busca um contexto leve sobre o tema. Primária: DuckDuckGo (web inteira); reserva: Wikipedia. Tudo fail-open: se vier vazio, gera sem contexto.

Infra desta peça grátis
Host
App Next.js (server) → APIs públicas
Rota/comando
src/lib/ddg.ts + searchWikipedia (dentro de /api/publish)
Código
searchTopic() em publish/route.ts
Conta/serviço
DuckDuckGo + Wikipedia REST (sem chave, sem cartão)
trafega: tema + contexto → prompt do Claude
4 · O texto — a voz da marca

Claude escreve a copy no tom @umpaisdemerda

O modelo gera um JSON: título-gancho, corpo, 3 insights (slides), CTA, legenda de IG e videoQueries (termos de footage). O marketBrief PT-BR (voz crua, apartidária, o "espelho" do servo voluntário) é injetado no prompt. Copy fica em cache por (tema, dia) → redisparo não repaga.

Infra desta peça pago · controle-gastos
Host
Anthropic API (nuvem)
Rota/comando
POST api.anthropic.com/v1/messages · modelo claude-haiku-4-5
Código
generateContent() em publish/route.ts + accounts.ts (marketBrief) + spend.ts (teto)
Conta/serviço
Anthropic · env ANTHROPIC_API_KEY — chave DEDICADA do produto no cofre .claude\chaves\umpaisdemerda.env (§1.3; feito em task_76ae412d)
o tema vira DUAS saídas visuais — carrossel (imagem) ou Reel (vídeo)

5B · Reel (vídeo)

Footage real + música + render

Busca footage de banco no tema, garante fundo (footage → ilustração → pular; nunca Reel preto), escolhe música por tema e renderiza o mp4 vertical.

8 · Registro, medição e freio

Grava o que saiu, mede e controla o gasto

Cada publicação é gravada (posts + livro-razão da vaga do dia, base da trava anti-dup). Um endpoint lê alcance/insights, e todo passo pago passa por um teto diário de gasto (bloqueia se estourar).

Conta/serviço

ciclo diário · token renovado mensalmente (refresh-token.yml → /api/refresh-token)
